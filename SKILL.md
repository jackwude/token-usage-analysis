# token-usage-analysis

查询和分析 OpenClaw / 阿里云百炼 API 的 Token 用量。

这个 Skill 默认面向**对话查询**，不是让用户记命令。

---

## 快速触发

以下说法都应视为可触发：

- `查 Token`
- `查 Token 用量`
- `Token 用量分析`
- `查过去 7 天 Token`
- `查最近 24 小时 Token`
- `查上周 Token`
- `查上周末 Token`
- `查 main 的 Token 用量`
- `哪个 Agent 用量最高`
- `最近哪天 Token 花得最多`
- `查 Token 消耗`

---

## 核心边界

- **只查询 OpenClaw / 阿里云百炼 API 的 Token 用量**
- **不查询 codexbar / Cursor 编辑器 Token**
- 数据来源于本地 session usage 快照日志
- 成本字段 `Cost` 依赖运行环境；若底层未提供，则可能为 `0` 或不可用

---

## 意图分流

收到请求后，优先判断属于哪一类：

### 1. 时间范围查询

适用于：
- `查过去 24 小时 Token`
- `查过去 7 天 Token`
- `查上周 Token`
- `查上周末 Token`
- `查 3 月 1 日到 3 月 7 日 Token`

默认支持：
- 过去 24 小时
- 过去 7 天
- 过去 30 天
- 上周末
- 上周
- 自定义日期范围

如果用户只说 `查 Token`，但没给时间范围：
直接让用户选择范围，不要擅自猜测。

### 2. Agent 维度查询

适用于：
- `查 main 的 Token 用量`
- `查 tg-office 最近 7 天 Token`
- `查各 Agent 用量`

若用户指定 Agent：
- 优先输出该 Agent 结果
- 若 Agent 不存在，提示可用 Agent 名称

若用户未指定 Agent：
- 默认统计所有 Agent
- 同时给出各 Agent 占比

### 3. 对比 / 排名查询

适用于：
- `哪个 Agent 用量最高`
- `各 Agent 用量占比`
- `最近 7 天谁最费 Token`

输出重点：
- 排名结果
- 占比
- 与第二名的差距（若明显）

### 4. 异常 / 洞察查询

适用于：
- `最近哪天 Token 花得最多`
- `哪天成本最高`
- `最近有没有异常增长`

输出重点：
- 峰值日期 / 峰值 Agent
- 是否存在明显突增
- 简短观察结论

---

## 执行流程

默认按以下顺序执行：

1. 识别用户是否已给时间范围
2. 若未给时间范围，先让用户选范围
3. 识别是否指定 Agent
4. 判断是普通汇总、对比排名，还是异常洞察
5. 调用分析脚本生成结果
6. 按固定模板输出
7. 若结果中存在明显峰值/异常，再补一句观察

---

## 输出要求

默认使用**固定四段式模板**，先给结论，再给明细，再给观察，最后给一句话判断。
只有当用户明确要求展开时，才允许在固定结构内增加更多细节；不要随意改模板结构。

### 固定输出模板

```markdown
📊 Token 用量分析（时间范围）

【结论】
- 总 Token：xxx
- 总 Cost：$xxx / 不可用
- 主消耗 Agent：main（xx.x%）
- 峰值日期：2026-03-09
- 异常提示：无明显异常 / xxx

【Agent 明细】
1. main
   - Token：xxx
   - Cost：$xxx / 不可用
   - Sessions：x
   - 占比：xx.x%

2. tg-office
   - Token：xxx
   - Cost：$xxx / 不可用
   - Sessions：x
   - 占比：xx.x%

【7 天趋势图】
03-07 | ███████████ 2.31M
03-08 | ████████████████████████████████ 7.11M
03-09 | ███████████████████████████ 6.17M
03-10 | █ 0.04M
03-11 | █ 0.12M

【关键观察】
- 观察 1
- 观察 2
- 观察 3

【一句话判断】
一句总结。
```

### 固化规则

1. 标题固定为：`📊 Token 用量分析（时间范围）`
2. 固定保留 5 个区块：`【结论】`、`【Agent 明细】`、`【7 天趋势图】`、`【关键观察】`、`【一句话判断】`
3. `【结论】` 区必须包含：总 Token、总 Cost、主消耗 Agent、峰值日期、异常提示
4. `【Agent 明细】` 区每个 Agent 必须包含：Token、Cost、Sessions、占比
5. `【7 天趋势图】` 默认输出按日汇总的文本柱状图，格式固定为：`MM-DD | 柱形 数值`
6. 柱状图优先展示 Token 每日变化；默认最多展示最近 7 个自然日
7. `【关键观察】` 默认写 **3 条**，尽量短，避免啰嗦
8. `【一句话判断】` 必须存在，用于快速扫读
9. 若无异常，`异常提示` 统一写：`无明显异常`
10. 若 Cost 不可用，统一写：`不可用`
11. 若只有一个 Agent，也保留 `【Agent 明细】` 区，不省略结构

---

## 降级处理

### 1. 没有日志数据

提示：
- 当前还没有足够的 Token 用量数据
- 可能是刚安装，或定时采集尚未积累完成

### 2. 指定时间范围无数据

提示：
- 该时间段内没有检测到有效数据
- 建议换一个更大的时间范围

### 3. 指定 Agent 不存在

提示：
- 当前未找到该 Agent
- 同时列出实际可用 Agent 名称

### 4. Cost 不可用

提示：
- Token 统计正常
- 成本字段当前不可用或底层未提供

## 底层实现

- `src/collector.py`：负责定时采集 session usage 快照
- `src/analyzer.py`：负责按时间范围汇总分析
- `install.sh`：负责安装脚本与定时任务

平时优先走对话查询；只有在安装、诊断或排障时，才需要命令行。

---

## 命令行（安装/排障用）

```bash
# 诊断状态
~/.openclaw/bin/collect-usage --diagnose

# 手动触发收集
~/.openclaw/bin/collect-usage

# 清理旧日志
~/.openclaw/bin/collect-usage --cleanup
```

---

## 当前局限

- 当前统计基于快照差分，不是逐事件精确记账
- 如果 session 中途重置、截断、跨天或日志缺失，结果可能存在误差
- 当前更擅长时间范围汇总，模型维度 / 异常诊断可继续增强
