# token-usage-analysis 输出模板

本文件定义 `token-usage-analysis` 的**固定结果输出模板**。

## 默认模板

```markdown
📊 Token 用量分析（时间范围）

【结论】
- 总 Token：16,106,551
- 总 Cost：$44.98
- 主消耗 Agent：main（97.4%）
- 峰值日期：2026-03-08
- 异常提示：2026-03-11 cost 疑似异常偏高

【Agent 明细】
1. main
   - Token：15,691,179
   - Cost：$43.55
   - Sessions：113
   - 占比：97.4%

2. tg-office
   - Token：415,372
   - Cost：$1.44
   - Sessions：8
   - 占比：2.6%

【关键观察】
- 过去 7 天用量高度集中在 main
- 3/7 ~ 3/9 是主要消耗区间
- 3/11 Token 不高但 Cost 偏高，建议排查底层 cost 口径

【一句话判断】
过去 7 天属于前高后低的集中消耗模式，主要开销来自 main。
```

## 固化规则

### 1. 标题
固定为：
- `📊 Token 用量分析（时间范围）`

### 2. 固定区块
必须固定输出以下 4 段：
1. `【结论】`
2. `【Agent 明细】`
3. `【关键观察】`
4. `【一句话判断】`

### 3. 结论区字段
必须包含：
- 总 Token
- 总 Cost
- 主消耗 Agent
- 峰值日期
- 异常提示

若无异常：
- 统一写 `无明显异常`

若 Cost 缺失：
- 统一写 `不可用`

### 4. Agent 明细区字段
每个 Agent 必须包含：
- Token
- Cost
- Sessions
- 占比

即使只有一个 Agent，也不要省略这个区块。

### 5. 关键观察
- 默认写 3 条
- 每条尽量短
- 优先写：集中度、峰值、异常
- 不要写成长段落

### 6. 一句话判断
- 必须存在
- 用于快速扫读
- 只保留 1 句结论

## 使用原则

- 默认用这套模板输出，不要临时改成 CLI 风格报表
- 用户要求展开时，只能在固定结构内补充，不要打乱区块顺序
- 优先保证一致性，其次再追求“文采”
