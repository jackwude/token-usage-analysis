# 审计模式

根据用户目标和可用材料，优先选择以下三种模式之一。

## 1. 快速审计

适用：
- 用户只想快速判断一个 skill 值不值得继续打磨
- 材料不完整
- 用户只要结论和 1~3 条建议

做法：
- 只读 SKILL.md 和关键资源
- 不强行补全所有分项
- 输出总体判断、最大短板、P0/P1 建议

## 2. 完整审计

适用：
- 用户要正式评分
- 用户准备发布 skill
- 用户要系统性优化

做法：
- 读 SKILL.md + scripts/ + references/ + assets/
- 必要时结合真实样本
- 按四维量表完整评分
- 输出完整审计报告

## 3. 对比审计

适用：
- 用户要比较多个 skill
- 用户要决定保留哪个 skill / 以哪个为主版本

做法：
- 对所有 skill 使用同一量表
- 重点看四维平均分、核心短板、P0 问题
- 不要只比总分

## 常见异常处理

### 没有真实案例
- 明确说明：当前结论主要基于静态文件，动态表现置信度有限。

### 用户只要优化动作，不要打分
- 可以跳过完整分数展示，但内部仍按四维判断后再给动作建议。

### skill 信息不足
- 先指出缺失项，再给有限结论；不要假装结论很稳。

### 用户只给一段对话，没有 skill 文件
- 先做动态表现复盘，再提示结论仅覆盖“实际表现”，不等于完整 skill 审计。
