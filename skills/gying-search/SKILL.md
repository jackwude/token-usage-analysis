# gying-search（观影网影视资源搜索）

## 快速触发

用户常见话术（任一命中即可执行）：
- `影视搜索：<关键词>` → 仅搜索，返回列表
- `gying 搜索：<关键词>` → 仅搜索，返回列表
- `找一下 <关键词> 的网盘资源` → 仅搜索，返回列表
- `帮我搜 <关键词>，要带评分` → 仅搜索，返回列表
- `搜 <关键词> 并转存第 X 个` → 搜索后转存指定链接
- `转存第 1、3、5 个` → （在搜索结果后）转存指定的链接

## 目标输出

**基础模式**（仅搜索）：
- 前 **5 条**（默认）网盘资源：**百度网盘优先**；若无百度网盘，则**退而返回夸克网盘**
- 返回字段：**资源名称** + 豆瓣评分 + 链接 + 发布者（如有）+ 更新时间（如有）
- 默认策略：**优先最新更新**；无法获取则按页面默认顺序
- **等待用户指令**，不自动转存
- 用户可指定数量：`影视搜索：XXX 给我 10 个`

**转存模式**（用户明确指定后）：
- 用户说"转存第 X 个"或"转存第 1、3、5 个"后
- 打开指定的百度网盘链接
- 转存到 `/影视资源（付想专用）` 文件夹
- 返回转存结果（成功/失败）

## 工作流（给 Agent 的操作步骤）

### 执行方式（默认用子代理跑，保持主会话不卡）

- **默认策略**：执行 gying 搜索/提取这类浏览器自动化时，优先交给**子代理**运行（后台跑完整流程+计时+排障），主会话只做：
  - 立即告知“已开始执行 + 预计耗时 + 会回传分阶段耗时与结果”
  - 等子代理完成后汇报最终结果
- **适用场景**：任何需要浏览器交互、可能弹窗/风控、或需要统计各阶段耗时的搜索任务
- **例外**：用户明确要求“就在主会话跑/边跑边问”时才在主会话直接执行

### 预备步骤：必须使用 gying 专用 Profile

在执行任何搜索/详情页提取前，先：
1. 运行 `bash ~/.openclaw/scripts/start-gying-chrome.sh`
2. **必须固定使用** `~/.openclaw/chrome-profiles/gying`
3. 打开 `https://www.gying.net/`
4. 先确认当前会话确实绑定到 `gying` 专用 Profile，再继续后续搜索；**不要**用默认浏览器上下文替代
5. 若已登录，优先在**首页搜索框**中执行搜索，不要默认依赖固定的旧搜索路由
6. 若未登录（落在 `/user/login`）：说明当前专用 Profile 尚未种入登录态；优先引导在这个专用 Profile 中完成一次登录，后续持续复用

**执行实现（重要，避免 Browser Relay 误解）**：
- 默认用 **Playwright 通过 CDP** 连接 `start-gying-chrome.sh` 启动的调试端口（通常 `127.0.0.1:9224`）来自动化。
- **不要**用 `browser` 工具的 `profile="chrome"` 路径来跑 gying（那条路径需要你在 Chrome 里点 **OpenClaw Browser Relay** 进行“附加标签页/Attach”，会引入额外不稳定因素）。
- 只有在你明确要求“接管你当前正在看的 Chrome 标签页”时，才会走 Browser Relay 附加流程。

> 这是一条强制前提：如果没有确认当前会话使用的是 `gying` 专用 Profile，不要继续执行搜索或把异常直接归因为“登录态失效”。先排查浏览器上下文，再判断是否真的掉登录态。

### 模式 A：仅搜索（默认）

1. 在专用 `gying` Profile 中打开 `https://www.gying.net/`
2. 若出现弹窗遮罩（拦截点击/输入），优先点击右下角 **「14天不再提醒」**；后续每次跳到**搜索页 / 详情页 / 网盘页**后，也先做同样检查，再继续操作
3. 使用首页搜索框（`input[type="search"][name="q"]`）输入关键词并回车执行搜索
4. **等待策略优化**：优先使用“条件等待”，不要默认固定 sleep
   - 搜索页：优先等待候选结果卡片、详情页链接（如 `/tv/` `/mv/` `/ac/`）或关键词命中文本出现
   - 详情页：优先等待标题、评分区、网盘 Tab、网盘链接区域出现
   - 只有在页面结构不稳定时，才加短暂兜底等待（例如 300-800ms），避免默认写死 1 秒以上固定等待
5. 在结果中选择**最匹配**的一条进入详情页，按下面顺序判断：
   - 优先标题完全匹配 / 高度匹配用户关键词
   - 类型一致优先（电影 / 剧 / 动漫）
   - 非广告 / 非推荐 / 非置顶干扰项优先
   - 信息更完整的详情页优先
   - 更新时间更近的结果优先
   - 详情页入口要同时识别：`/tv/`、`/mv/`、`/ac/`，不要只依赖单一路径
   - **如果没有完全关键词匹配的结果，不要直接判定“没找到”**；应先返回搜索结果前 **5** 个候选（标题/类型/评分/可见信息），让用户选择后再进入详情页
   - **候选抽取规则（重要）**：搜索结果里同一条内容通常会出现两次链接（封面链接 + 标题链接），要按 `href` 去重；若标题链接文本为空或过短，不要丢弃，应继续从其 `parent / grandparent / card` 容器文本中提取标题、类型、评分等可见信息
6. 在详情页：
   - 提取豆瓣评分（没有则写"暂无评分"）
   - **先确认已进入「网盘下载 / 网盘资源」区域**，不要只因标题加载成功就认为详情页已准备完成
   - 先读取当前页面可见的网盘分组与数量（如：迅雷网盘 / 百度网盘 / 夸克网盘 / 阿里网盘），尤其关注是否出现 **`百度网盘 <数字>`**
   - **注意：默认展示的分组可能是迅雷网盘**；这不代表百度/夸克资源不存在，必须继续检查分组切换区
   - 再按优先级切换分组：**百度网盘 → 夸克网盘**
   - **重要修正（避免误判）**：判断“百度是否存在”的信号，应基于“进入网盘资源区后的整页正文/HTML + 分组计数”，而不是仅基于你当前停留的某个分组（例如已切到夸克后再判断会误判百度不存在）
   - 分组切换不要只依赖单一 `get_by_role`；应结合文本匹配、普通链接选择器、容器内点击（JS fallback）等多种方式兜底
   - **切换生效验证（必须）**：
     - 切到“百度网盘”后，必须验证页面中 `a[href*="pan.baidu.com"]` 的数量从 0 变为 >0（或至少明显增加）；否则视为“切换未生效”，需要重试切换/滚动资源区后再试
     - 切到“夸克网盘”后，验证 `a[href*="pan.quark.cn"]`
7. 抽取前 N 条，按下面优先级选择：
   - **优先 `pan.baidu.com`（百度网盘）**
   - **只有在确认“百度资源确实不存在”后，才允许退而选择 `pan.quark.cn`（夸克网盘）`**
   - 若正文/HTML/分组统计里已经出现 `百度网盘`、`百度网盘 79` 之类计数、或 `pan.baidu.com`，就**不能**直接判定“没有百度资源”；必须先进入 **百度二次抽取模式**：
     1. 重新定位“网盘下载 / 网盘资源”区域
     2. **即使未成功点击到“百度网盘”分组，也先直接全页扫描一次 `pan.baidu.com` 链接**（很多页面链接已在 DOM/HTML 中存在）
     3. 在网盘资源区内重新查找并切换“百度网盘”分组（多路选择器兜底）
     4. 重扫该区域容器文本 / HTML / 链接，必要时滚动资源区后再抽取一次
     5. 只有二次抽取仍失败，才能标注为“页面显示存在百度资源，但当前自动提取未成功”
   - 只有在**百度确实不存在**或**百度二次抽取明确失败**后，才可退回夸克网盘
   - 同时提取发布者/更新时间（如果页面给出，用于"优先最新"）
     - **推荐解析规则**：通常这两个字段位于“该条资源所在行/卡片文本”的末尾
       - 更新时间常见形态：`今天/昨天/前天/7天前/10月前/1年前/2026-03-10` 等
       - 发布者通常是紧挨着更新时间前的那一段（可按“行尾倒数第 2 个 token/词组”取值）；拿不准就填 `未知`
8. **抽取失败时不要立刻判定“无资源”**：
   - 先做二次核查：检查正文/HTML 中是否已出现 `百度网盘` / `夸克网盘` / `pan.baidu.com` / `pan.quark.cn`
   - 若这些关键词已出现，说明更可能是“分组未切对”或“结构识别失败”，应继续排查，而不是直接结束任务
   - **尤其是百度网盘**：如果正文/HTML/分组统计显示百度存在，就不能直接说“未找到百度网盘”，只能说：`页面显示存在百度网盘资源，但当前自动提取未成功`（若后续返回夸克结果，也要明确这是兜底结果，而不是百度不存在）
   - 只有在完成二次核查后，才能下结论：`确实无百度/夸克资源` 或 `页面结构异常，暂未提取成功`
9. 输出为固定格式的清晰列表；若网站/页面结构异常：按降级策略说明原因
   - 默认使用“**资源名称直接作为超链接**”的格式，链接地址即实际提取到的网盘链接
   - 若标题包含过多特殊字符、导致 Telegram Markdown 转义风险较高，可退回普通文本格式，避免整条消息发送异常
10. **收尾（必须做）**：任务完成后，关闭本次打开的所有非必要页面/标签页（搜索页、详情页、网盘页等），避免浏览器长期堆积占用内存；**但前提是任务已完成，或已明确进入“无资源/结构异常”的最终状态，禁止在中间排查阶段提前关页**

**固定输出模板：**

```markdown
🎬 影视资源搜索结果：《关键词》

1. [资源名称 xxx](<链接>)
   豆瓣评分：8.2 / 暂无评分
   发布者：xxx / 未知
   更新时间：2026-03-10 / 今天 / 未知
```

缺失字段时统一写法：
- 没评分 → `暂无评分`
- 没更新时间 → `未知`
- 没有百度网盘但有夸克网盘 → 直接返回夸克网盘结果，并标注 `（未找到百度网盘，已退回夸克网盘）`
- 百度网盘和夸克网盘都没有 → 明确提示用户 `当前结果未找到百度网盘或夸克网盘资源，建议换个搜索词`

### 模式 B：搜索 + 转存（用户指定）

1. 先执行**模式 A**搜索到链接列表，**展示给用户**
2. **等待用户明确指令**（如"转存第 1、3、5 个"）
3. 对用户指定的每个链接：
   - 用 browser 打开 `https://pan.baidu.com/s/<share_id>?pwd=<提取码>`
   - 等待页面加载（约 3-5 秒）
   - 找到并点击 **"保存到网盘"** 按钮（触发文件夹选择弹窗）
   - **关键步骤：选择目标文件夹**
     - 在文件夹选择弹窗中，找到 **"影视资源（付想专用）"** 文件夹
     - 点击该文件夹（会展开显示子文件夹）
     - 点击 **"确定"** 按钮确认选择
   - 等待转存成功提示
   - 关闭当前标签页，继续下一个链接
4. 返回转存结果汇总（成功/失败数量）

> 重点说明：观影网本身优先依赖 `gying` 专用 Profile 保存登录态；百度网盘部分是否长期保 Profile 视后续稳定性再决定。

**注意事项：**
- 必须点击"保存到网盘"按钮后才会弹出文件夹选择器
- 文件夹选择器中需要找到并点击"影视资源（付想专用）"，不能直接点确定
- 每个链接转存后关闭标签页，避免打开过多页面

> 页面结构变化/提取失败时：读取 `references/gying-ui.md` 排查。
> 转存配置详情：读取 `references/baidu-save-config.md`。

## 降级策略（必须显式标注）

- **站点不可访问/加载失败**：说明无法访问 gying.net，建议稍后重试。
- **专用 Profile 未登录**：明确说明当前 `gying` Profile 落在登录页，需要先在该专用 Profile 中完成一次登录。
- **旧搜索路由失效 / 固定搜索 URL 返回 404**：改为从已登录首页的搜索框执行搜索，并标注“站点搜索路由已变化，已切换为首页搜索框模式”。
- **搜索结果中没有完全关键词匹配内容**：不要误进相似但错误的详情页；先返回前 5 个候选结果让用户选择，再继续后续详情页提取。
- **无法定位百度网盘 Tab 或链接列表**：改为返回你在页面能看到的网盘链接（并标注"结构变化，可能不完整"）。
- **无法验证链接有效性**：默认不做逐条有效性验证；若用户明确要求"只要有效链接"，再做逐条打开确认（会变慢）。

## 可选参数

- 返回数量：用户说"给我前 5/10/20 个"则按数量返回；**未说则默认 5 个**。
