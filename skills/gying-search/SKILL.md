# gying-search（观影网影视资源搜索）

## 快速触发

用户常见话术（任一命中即可执行）：
- `影视搜索：<关键词>` → 仅搜索，返回列表
- `gying 搜索：<关键词>` → 仅搜索，返回列表
- `找一下 <关键词> 的网盘资源` → 仅搜索，返回列表
- `帮我搜 <关键词>，要带评分` → 仅搜索，返回列表
- `搜 <关键词> 并转存第 X 个` → 搜索后转存指定链接
- `转存第 1、3、5 个` → （在搜索结果后）转存指定的链接

## 目标输出

**基础模式**（仅搜索）：
- 前 **5 条**（默认）网盘资源：**百度网盘优先**；若无百度网盘，则**退而返回夸克网盘**
- 返回字段：标题 + 豆瓣评分 + 链接 + 提取码（如有）+ 更新时间（如有）
- 默认策略：**优先最新更新**；无法获取则按页面默认顺序
- **等待用户指令**，不自动转存
- 用户可指定数量：`影视搜索：XXX 给我 10 个`

**转存模式**（用户明确指定后）：
- 用户说"转存第 X 个"或"转存第 1、3、5 个"后
- 打开指定的百度网盘链接
- 转存到 `/影视资源（付想专用）` 文件夹
- 返回转存结果（成功/失败）

## 工作流（给 Agent 的操作步骤）

### 预备步骤：必须使用 gying 专用 Profile

在执行任何搜索/详情页提取前，先：
1. 运行 `bash ~/.openclaw/scripts/start-gying-chrome.sh`
2. **必须固定使用** `~/.openclaw/chrome-profiles/gying`
3. 打开 `https://www.gying.net/`
4. 先确认当前会话确实绑定到 `gying` 专用 Profile，再继续后续搜索；**不要**用默认浏览器上下文替代
5. 若已登录，优先在**首页搜索框**中执行搜索，不要默认依赖固定的旧搜索路由
6. 若未登录（落在 `/user/login`）：说明当前专用 Profile 尚未种入登录态；优先引导在这个专用 Profile 中完成一次登录，后续持续复用

> 这是一条强制前提：如果没有确认当前会话使用的是 `gying` 专用 Profile，不要继续执行搜索或把异常直接归因为“登录态失效”。先排查浏览器上下文，再判断是否真的掉登录态。

### 模式 A：仅搜索（默认）

1. 在专用 `gying` Profile 中打开 `https://www.gying.net/`
2. 使用首页搜索框（`input[type="search"][name="q"]`）输入关键词并回车执行搜索
3. 在结果中选择**最匹配**的一条进入详情页，按下面顺序判断：
   - 标题完全匹配优先
   - 类型一致优先（电影 / 剧 / 动漫）
   - 非广告 / 非推荐 / 非置顶干扰项优先
   - 信息更完整的详情页优先
   - 更新时间更近的结果优先
   - 如果仍有歧义，返回最可能结果，并标注“可能存在同名结果混淆”
4. 在详情页：
   - 提取豆瓣评分（没有则写"暂无评分"）
   - 切到「百度网盘」Tab（或包含 `pan.baidu.com` 的区域）
5. 抽取前 N 条，按下面优先级选择：
   - **优先 `pan.baidu.com`（百度网盘）**
   - 若当前条目没有百度网盘，则**退而选择 `pan.quark.cn`（夸克网盘）**
   - 若百度网盘和夸克网盘都没有，再明确提示用户：当前结果没有百度/夸克网盘资源，建议换个搜索词
   - 同时提取提取码（如果页面给出）
   - 同时提取更新时间/发布信息（如果页面给出，用于"优先最新"）
6. 输出为固定格式的清晰列表；若网站/页面结构异常：按降级策略说明原因

**固定输出模板：**

```markdown
🎬 影视资源搜索结果：《关键词》

1. 标题：xxx
   豆瓣评分：8.2 / 暂无评分
   百度网盘：<链接>
   提取码：1234 / 无
   更新时间：2026-03-10 / 未知
```

缺失字段时统一写法：
- 没评分 → `暂无评分`
- 没提取码 → `无`
- 没更新时间 → `未知`
- 没有百度网盘但有夸克网盘 → 直接返回夸克网盘结果，并标注 `（未找到百度网盘，已退回夸克网盘）`
- 百度网盘和夸克网盘都没有 → 明确提示用户 `当前结果未找到百度网盘或夸克网盘资源，建议换个搜索词`

### 模式 B：搜索 + 转存（用户指定）

1. 先执行**模式 A**搜索到链接列表，**展示给用户**
2. **等待用户明确指令**（如"转存第 1、3、5 个"）
3. 对用户指定的每个链接：
   - 用 browser 打开 `https://pan.baidu.com/s/<share_id>?pwd=<提取码>`
   - 等待页面加载（约 3-5 秒）
   - 找到并点击 **"保存到网盘"** 按钮（触发文件夹选择弹窗）
   - **关键步骤：选择目标文件夹**
     - 在文件夹选择弹窗中，找到 **"影视资源（付想专用）"** 文件夹
     - 点击该文件夹（会展开显示子文件夹）
     - 点击 **"确定"** 按钮确认选择
   - 等待转存成功提示
   - 关闭当前标签页，继续下一个链接
4. 返回转存结果汇总（成功/失败数量）

> 重点说明：观影网本身优先依赖 `gying` 专用 Profile 保存登录态；百度网盘部分是否长期保 Profile 视后续稳定性再决定。

**注意事项：**
- 必须点击"保存到网盘"按钮后才会弹出文件夹选择器
- 文件夹选择器中需要找到并点击"影视资源（付想专用）"，不能直接点确定
- 每个链接转存后关闭标签页，避免打开过多页面

> 页面结构变化/提取失败时：读取 `references/gying-ui.md` 排查。
> 转存配置详情：读取 `references/baidu-save-config.md`。

## 降级策略（必须显式标注）

- **站点不可访问/加载失败**：说明无法访问 gying.net，建议稍后重试。
- **专用 Profile 未登录**：明确说明当前 `gying` Profile 落在登录页，需要先在该专用 Profile 中完成一次登录。
- **旧搜索路由失效 / 固定搜索 URL 返回 404**：改为从已登录首页的搜索框执行搜索，并标注“站点搜索路由已变化，已切换为首页搜索框模式”。
- **无法定位百度网盘 Tab 或链接列表**：改为返回你在页面能看到的网盘链接（并标注"结构变化，可能不完整"）。
- **无法验证链接有效性**：默认不做逐条有效性验证；若用户明确要求"只要有效链接"，再做逐条打开确认（会变慢）。

## 可选参数

- 返回数量：用户说"给我前 5/10/20 个"则按数量返回；**未说则默认 5 个**。

## 兼容的命令式用法（调试用）

> `search.sh` 当前定位为**调试辅助脚本**：用于快速生成搜索入口和操作提示，方便人工/Agent 调试；它**不是**完整的自动搜索与提取引擎。

```bash
bash ~/.openclaw/scripts/start-gying-chrome.sh
bash ~/.openclaw/workspace/skills/gying-search/search.sh <关键词> [数量]
```
